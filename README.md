# Книжная Лавка Студента

Сайт книжного магазина в Новороссийске с каталогом направлений, рецензиями, событиями книжного клуба и интеграцией с Sanity CMS.

Проект сделан как полноценный frontend-кейс для портфолио: с серверным рендерингом на Next.js, динамическим контентом из CMS, адаптивной версткой, SEO-метаданными, страницами для пользовательских сценариев и отдельной админ-зоной Sanity Studio.

## Демо

Production: https://lavka-studenta-bookstore.ru

## Основные возможности

* Главная страница книжного магазина с секциями о магазине, книжном клубе, подборках, подарках и контактах.
* Страница рецензий и книжных рекомендаций.
* Страница книжного клуба и событий.
* Динамический контент из Sanity CMS.
* Встроенная админ-панель Sanity Studio по маршруту `/admin`.
* SEO-метаданные и Open Graph-изображение.
* Адаптивная верстка для desktop/tablet/mobile.
* Декоративные UI-эффекты и анимации.
* Безопасная работа с env-переменными через `.env.example`.

## Работа с данными

Контент для рецензий и событий хранится в Sanity CMS.

В проекте есть отдельный слой для работы с CMS:

* `sanity/client.ts` — создание Sanity client;
* `sanity/env.ts` — чтение и проверка env-переменных;
* `sanity/queries.ts` — GROQ-запросы;
* `sanity/fetchers.ts` — функции получения данных;
* `sanity/schemaTypes/` — схемы документов Sanity.

Если Sanity env-переменные не заданы, проект не падает, а возвращает безопасные fallback-значения. Это позволяет запускать проект локально и проверять сборку без доступа к production CMS.

## Что было технически важным

### 1. Интеграция Next.js и Sanity CMS

Проект использует Sanity как headless CMS. Контент можно редактировать через Studio, а frontend получает опубликованные данные через отдельные fetcher-функции.

### 2. Разделение статического и динамического контента

Часть контента хранится в коде как стабильные данные магазина, а рецензии и события вынесены в CMS. Это позволяет не пересобирать интерфейс вручную при каждом изменении контента.

### 3. App Router и серверные компоненты

Главная страница получает данные на сервере и отдает уже подготовленный HTML. Это улучшает первичную загрузку и SEO.

### 4. SEO и социальные превью

В проекте настроены базовые metadata, canonical URL, Open Graph и Twitter card. Также используется отдельное Open Graph-изображение.

### 5. Адаптивный интерфейс

Верстка рассчитана на разные размеры экрана: от мобильных устройств до desktop. Особое внимание уделено визуальной подаче, типографике, отступам и состояниям интерфейса.

## Локальный запуск

### 1. Клонировать репозиторий

```bash
git clone https://github.com/MariaSemenovaDev/lavka-studenta-bookstore.git
cd lavka-studenta-bookstore
```

### 2. Установить зависимости

```bash
npm install
```

### 3. Создать env-файл

Создайте файл `.env.local` на основе `.env.example`:

```bash
cp .env.example .env.local
```

Пример переменных:

```env
NEXT_PUBLIC_SANITY_PROJECT_ID=
NEXT_PUBLIC_SANITY_DATASET=production
NEXT_PUBLIC_SANITY_API_VERSION=2025-01-01
NEXT_PUBLIC_SITE_URL=https://lavka-studenta-bookstore.ru
```

### 4. Запустить проект

```bash
npm run dev
```

Проект будет доступен по адресу:

```txt
http://localhost:3000
```
